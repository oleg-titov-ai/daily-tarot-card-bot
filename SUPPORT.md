# Support

This repository is maintained as a public automation and portfolio project.

## Getting Help

Before opening an issue:

1. read the README;
2. review the legal disclaimer;
3. check the setup checklist;
4. review demo screenshots;
5. make sure only demo data is used in public examples.

## What to Include in an Issue

- short problem description;
- steps to reproduce;
- expected behavior;
- actual behavior;
- whether the issue is related to Telegram, n8n, PostgreSQL, or scheduling;
- safe logs with private data removed.

## What Not to Include

Do not include Telegram tokens, real chat IDs, real user IDs, private messages, n8n credentials, production webhook URLs, database passwords, or `.env` files.

## Privacy Note

A production version should collect only minimum required user data and provide an unsubscribe path.

## Project Status

This is not a commercial support channel. Issues and improvements are handled as time allows.
