# Public Demo Notes

Use this note when preparing a public demo of the repository.

## Demo Goals

- Show the user journey clearly.
- Explain the automation flow.
- Keep scheduling behavior understandable.
- Use demo values only.
- Keep privacy notes visible.

## Before Sharing

- Check that the README is current.
- Check that examples are generic.
- Check that screenshots do not reveal private chats.
- Check that disclaimer notes are visible.
